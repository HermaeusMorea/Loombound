# Style Bible

## System Tone

系统口吻应当像一位过度严肃、坚持礼序、却略带荒诞感的礼官。重点不是搞笑，而是以严正语气处理并不完全值得如此郑重的小事。

## Writing Constraints

- 句式简洁，不写长段说教
- 尽量使用“礼、序、名分、克己、谦退、避争、失仪、僭越、守分”等词汇
- 荒诞感来自“过度庄重”，而不是现代段子或吐槽
- 文案不能掩盖规则判断
- 文案与规则结论冲突时，以规则结论为准

## Narration Functions

旁白文本可拆为三类：

1. `opening`: 解释当前局势被视作何种礼制问题
2. `judgement`: 宣告当前生效规则
3. `warning`: 对违礼行为给出记录或警示

## Examples Of Good Tone

- “此处不贵强取，贵在知止。”
- “重器并陈，不宜先逐浮华。”
- “名分既列，当先定其序，而后言得失。”

## Examples To Avoid

- 现代互联网口吻
- 明显 meme 化表达
- 明显阴阳怪气
- 把系统写成 omniscient strategist

## Naming Guidance

- 数据字段命名保持清晰、直白、英文
- 规则 `id` 稳定、可追踪
- 文案模板名称表达用途，不表达情绪
